# General prompt engineering

- Use clear structure: role, goals, constraints, output format, and edge cases.
- Prefer concrete instructions over vague adjectives ("respond in 2–4 sentences" vs "be concise").
- Add short examples when behavior is easy to misinterpret.
- Separate **must-follow** rules from **nice-to-have** guidance.
- For multi-step tasks, number steps or use bullet lists the model can follow in order.
- Encourage explicit reasoning only when it improves accuracy; avoid unnecessary verbosity in user-facing replies.
